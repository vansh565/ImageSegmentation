# 🎯 Problem Statement

Autonomous drones require the ability to understand their surrounding environment before performing critical tasks such as landing, navigation, inspection, and surveillance. Traditional computer vision techniques rely on handcrafted image processing algorithms that struggle under varying illumination, shadows, seasonal changes, and complex environments.

For safe autonomous landing, the drone must distinguish between **safe landing surfaces** such as grass or open fields and **unsafe regions** including buildings, trees, roads, vehicles, people, and water bodies.

Object detection alone is insufficient because it predicts only bounding boxes, while a landing system requires **pixel-level understanding** of the environment.

Semantic segmentation solves this problem by assigning every pixel in an image to a semantic category, enabling the drone to understand the complete scene before making landing decisions.

This project focuses on building an efficient semantic segmentation framework capable of running on edge devices while maintaining high segmentation accuracy and real-time inference performance.

# 💡 Motivation

Modern autonomous UAV systems operate in highly dynamic outdoor environments where landing conditions constantly change.

Examples include:

- Disaster response missions
- Precision agriculture
- Search and rescue
- Infrastructure inspection
- Autonomous delivery drones
- Environmental monitoring

In these scenarios, manually selecting landing locations is either impossible or unsafe.

The objective of this project is to develop a lightweight semantic segmentation model capable of understanding aerial scenes in real time while remaining computationally efficient enough for deployment on embedded hardware such as NVIDIA Jetson, Raspberry Pi, or edge AI accelerators.

The project combines deep learning, model optimization, and computer vision techniques to bridge the gap between research-grade semantic segmentation models and practical real-world deployment.

# 🧠 Semantic Segmentation Explained

Unlike image classification, which predicts a single label for the entire image, or object detection, which predicts bounding boxes around objects, semantic segmentation performs **dense prediction**.

Every pixel inside the image receives its own class label.

Example:

Original Image

Grass       Road       Tree

↓

Semantic Segmentation

111111111111111

111111111111111

222222222222222

333333333333333

Where

1 → Grass

2 → Road

3 → Tree

As a result, the drone understands the complete geometry of the surrounding environment rather than isolated objects.

# 📖 Knowledge Distillation

Deep neural networks with millions of parameters generally provide high segmentation accuracy but are computationally expensive.

Knowledge Distillation transfers the knowledge learned by a large **Teacher Network** into a lightweight **Student Network**.

Teacher Model
        │
        ▼
Produces Soft Labels
        │
        ▼
Student Model
        │
        ▼
Learns Teacher Knowledge

Advantages:

- Smaller model size
- Faster inference
- Lower memory usage
- Better deployment on embedded hardware
- Minimal accuracy degradation

The student model learns not only from the ground-truth labels but also from the probability distributions generated by the teacher network.


# ⚡ Dynamic Quantization

Deep learning models normally store weights using 32-bit floating-point numbers.

Dynamic Quantization converts these weights into 8-bit integers.

FP32 Weights

↓

INT8 Weights

Benefits

- Smaller model size
- Lower RAM usage
- Faster inference
- Better CPU performance
- Improved edge deployment

Model Size

Teacher Model : 45 MB

Student Model : 8 MB

Quantized Model : 3 MB


# 📂 Dataset Description

The project uses the **Semantic Drone Dataset**, containing aerial images captured from low-altitude UAVs.

Dataset Statistics

- 400 RGB Images
- 400 Pixel-wise Annotation Masks
- Resolution up to 6000×4000 pixels
- 23 Semantic Classes

Each image is manually annotated, assigning every pixel to one of the predefined semantic classes.

# 🏗️ MobileNetV2-UNet Architecture

The segmentation model combines two powerful architectures.

## Encoder

The encoder is based on **MobileNetV2**, which extracts hierarchical image features using depthwise separable convolutions.

Advantages

- Lightweight
- High accuracy
- Fast inference
- Low computational cost

The encoder progressively reduces spatial resolution while increasing feature depth.

Input

256×256×3

↓

128×128×32

↓

64×64×64

↓

32×32×128

↓

16×16×320

↓

8×8×1280

---

## Decoder

The decoder reconstructs high-resolution segmentation maps.

It performs:

- Transposed Convolution
- Feature Upsampling
- Skip Connections
- Feature Fusion

Output

256×256×23

where each channel represents the probability of one semantic class.


# 🔄 Skip Connections

UNet introduces skip connections between encoder and decoder layers.

Encoder Features
        │
        │──────────────┐
        ▼              │
 Bottleneck            │
        ▼              │
 Decoder ◄─────────────┘

Skip connections preserve fine spatial details that would otherwise be lost during downsampling.

Benefits

- Better boundary detection
- Sharper segmentation masks
- Improved localization
- Faster convergence


# 🌿 Landing Zone Detection

After semantic segmentation, the project converts the multi-class segmentation map into a binary landing map.

Input

23-Class Segmentation

↓

Extract Grass Class

↓

Binary Mask

Grass → 1

Others → 0

↓

Morphological Filtering

↓

Connected Component Analysis

↓

Largest Safe Region

↓

Landing Zone Candidate

This binary representation allows autonomous drones to identify potential landing areas while rejecting unsafe regions.
